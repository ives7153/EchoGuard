## 硬件接线与自检文档（ESP32-S3-DevKitC-1 N8R8 + Ra-02 + 传感器）

本文档用于指导 ESP32-S3-DevKitC-1 N8R8 开发板连接 Ra-02 LoRa 模块、SHT20 温湿度传感器、MPU6050 姿态传感器、MQ-135 气体传感器以及可选状态 LED。节点侧负责 WiFi CSI、传感器采集与 LoRa 上报，Gateway 侧负责 LoRa 接收与串口转发。

所有模块必须共地。Ra-02、SHT20、MPU6050 推荐使用 3.3V 供电与 3.3V 逻辑电平；MQ-135 可使用 5V 供电，但其模拟输出进入 ESP32-S3 ADC 前必须确认不超过 3.3V。

### 实物接线图文字描述

#### ESP32-S3-DevKitC-1 N8R8 主控板

开发板通过 USB 连接电脑，用于供电、烧录固件和查看串口日志。
节点和 Gateway 均以 ESP32-S3 作为主控，所有外设的 GND 都需要连接到开发板 GND。
接线前建议先确认板卡丝印、实际可用 GPIO 与固件中的引脚定义一致。

#### Ra-02 LoRa 模块

Ra-02 的 VCC 接 ESP32-S3 的 3V3，GND 接公共地，禁止接入 5V。
SPI 接线为 NSS 接 GPIO10、SCK 接 GPIO12、MOSI 接 GPIO11、MISO 接 GPIO13；控制线 DIO0 接 GPIO4，RST 接 GPIO21。
LoRa 天线必须在上电和发射前安装好，模块尽量远离 USB 线、电源线和传感器加热区域。

#### SHT20 温湿度传感器

SHT20 使用 I2C 总线，默认地址为 0x40，VCC 接 3V3，GND 接公共地。
SDA 接 GPIO8，SCL 接 GPIO9，与 MPU6050 共用同一组 I2C 线。
如果模块板没有自带上拉电阻，需要在 SDA/SCL 到 3V3 之间各加一个 4.7kΩ 上拉电阻。

#### MPU6050 姿态传感器

MPU6050 的 VCC 推荐接 3V3，GND 接公共地，避免 I2C 电平超过 ESP32-S3 可接受范围。
SDA 接 GPIO8，SCL 接 GPIO9，与 SHT20 并联在同一条 I2C 总线上。
安装时尽量固定模块方向，避免线缆晃动导致姿态数据抖动。

#### MQ-135 气体传感器

MQ-135 模块通常使用 5V 给加热丝供电，GND 必须与 ESP32-S3 共地。
模拟输出 AO 接 GPIO6，但接入前应先测量 AO 最大电压，确保不会超过 3.3V。
如果 AO 可能输出 5V，需要增加电阻分压或使用电平兼容的传感器模块。

#### 状态 LED

状态 LED 为可选模块，可分别连接 GPIO15 和 GPIO16，用于指示 WiFi、LoRa 或 CSI 工作状态。
每个 LED 必须串联 220Ω-1kΩ 限流电阻，LED 负极接 GND。
如果点亮逻辑与预期相反，可在固件中调整 GPIO 输出高低电平。

#### Gateway USB 串口

Gateway 通过 ESP32-S3 开发板 USB 口连接上位机，串口波特率建议设置为 115200。
上电后应在串口工具中看到启动日志、LoRa 初始化日志和接收数据输出。
若串口无法识别，优先更换数据 USB 线并检查驱动和端口号。

### 5个最常见硬件坑点及解决办法

| 坑点 | 典型现象 | 解决办法 |
|------|----------|----------|
| Ra-02 误接 5V | LoRa 模块发热、无法初始化，严重时损坏 | Ra-02 VCC 只能接 3V3；上电前用万用表确认供电电压 |
| LoRa 未接天线就发射 | 通信距离极短、模块异常发热、收发不稳定 | 使用匹配频段天线，上电发射前必须接好天线 |
| SPI 线接反或接触不良 | 串口提示 LoRa 初始化失败，收不到包 | 逐根核对 NSS/SCK/MOSI/MISO/DIO0/RST，缩短杜邦线并压紧插接点 |
| I2C 缺少上拉或地址异常 | 扫描不到 SHT20/MPU6050，读数超时或全 0 | 给 SDA/SCL 加 4.7kΩ 上拉到 3V3，使用 I2C 扫描程序确认地址 |
| MQ-135 AO 超过 ADC 电压范围 | ADC 读数异常，严重时损坏 ESP32-S3 | 测量 AO 电压；高于 3.3V 时增加分压电路或更换 3.3V 兼容模块 |

### 电源与天线布局建议

1. 主电源建议使用稳定 5V 供电，例如质量可靠的 USB 适配器、充电宝或调试电源，避免电压下陷导致 ESP32-S3 重启。
2. ESP32-S3、LoRa、传感器必须共地，建议使用较短且较粗的 GND 线，减少瞬态电流造成的地线压降。
3. Ra-02 模块供电旁建议增加 10uF 与 0.1uF 去耦电容，电容尽量靠近模块 VCC/GND。
4. MQ-135 加热丝电流较大，建议从 5V 电源入口单独分出供电线，不要与 LoRa 或 I2C 传感器共用细长供电线。
5. LoRa 天线应远离 ESP32-S3 WiFi 天线、USB 线、MQ-135 加热区域和大电流电源线。
6. LoRa 天线与 WiFi 天线尽量保持 5-10 cm 以上距离，避免平行紧贴或被金属外壳遮挡。
7. Ra-02 射频端应朝向设备外侧，若需要安装外壳，优先使用塑料外壳并让天线外露或靠近外壳边缘。
8. SPI 线尽量短，建议小于 15 cm；I2C 线也应避免过长，必要时降低 I2C 速率或改善上拉。

### 硬件上电自检清单

#### 节点上电自检

| 检查项 | 正常 LED/串口现象 | 异常时优先检查 |
|--------|-------------------|----------------|
| 开发板供电 | 开发板电源灯亮，USB 串口可识别 | USB 线、电源开关、5V 输入、GND 是否连接 |
| 固件启动 | 串口 115200 可看到启动日志和节点初始化信息 | 串口号、波特率、烧录是否成功、启动模式 |
| LoRa 初始化 | 串口显示 LoRa/SX1278 初始化成功，状态 LED 可按固件逻辑闪烁 | Ra-02 供电、SPI 接线、RST/DIO0、天线 |
| I2C 传感器 | 串口可看到 SHT20 与 MPU6050 初始化成功或扫描到对应地址 | SDA/SCL 是否接反、上拉电阻、模块地址、3V3 供电 |
| SHT20 读数 | 串口温湿度接近环境值，数值连续变化 | I2C 地址、供电稳定性、传感器接触 |
| MPU6050 读数 | 静止时加速度约为 1g，移动模块时数据有响应 | 模块方向、焊点虚接、I2C 线过长 |
| MQ-135 读数 | 上电预热后 ADC 数值缓慢稳定 | AO 是否超过 3.3V、是否共地、分压电路 |
| WiFi/CSI 状态 | WiFi/CSI 指示 LED 按固件逻辑闪烁，串口输出 CSI 或 WiFi 状态 | WiFi 天线、供电压降、固件配置 |
| LoRa 上报 | 串口显示周期性发送数据或发送成功状态 | Gateway 是否在线、频率/扩频因子/带宽配置是否一致 |
